# Mo7ammed Abuzaid — Photographer Portfolio

Scroll-driven, editorial/minimal photographer portfolio. Built **static-first**
with an abstracted data layer so it can move to a CMS (Sanity) later without
touching any UI.

**Stack:** Next.js 16 (App Router) · TypeScript · Tailwind v4 · GSAP +
ScrollTrigger · Lenis (smooth scroll) · Motion (Framer Motion) · `sharp` (dev,
placeholder generation). Deploy target: Vercel.

## Getting started

```bash
npm install
npm run gen:assets   # generate placeholder images / frames / blur map into /public
npm run dev          # http://localhost:3000
npm run build        # production build (typechecks + static prerender)
```

## Architecture

```
src/
  app/            layout (fonts + providers) + page (server: awaits data, renders sections)
  lib/
    data/index.ts  ← the ONLY data boundary: getProjects/getServices/getPackages/getSiteSettings
    types.ts       Project · MediaItem · Service · Package · SiteSettings
    blur.ts        re-exports the generated LQIP map
  data/           static source content (projects, services, packages, siteSettings)
  components/
    providers/    SmoothScrollProvider (Lenis+GSAP), CursorProvider
    cursor/ ui/ layout/   CustomCursor, MagneticButton, MediaImage, VideoTile, SectionShell, StickyNav, Preloader
  hooks/          useIsTouch, usePrefersReducedMotion, useFrameSequence, useGsapContext
  sections/       the 11 sections
scripts/generate-placeholders.mjs   placeholder pipeline
public/images · public/frames · public/videos
```

### Data layer (the contract)

UI components import data **only** through `@/lib/data` and types from
`@/lib/types` — never from `src/data/*` directly. To migrate to Sanity later,
rewrite the four function bodies in [`src/lib/data/index.ts`](src/lib/data/index.ts)
(e.g. GROQ fetches). Nothing in `src/sections` or `src/components` changes.

### Sections

Hero (split-name + hover-cycle portrait) · About (frame-by-frame scrub, pinned) ·
Selected Work (3D depth loop) · Projects (center-focus scroll) · Featured
(grow-to-fill) · Video Work (vertical→horizontal, play cursor) · Editorial
(scattered + parallax) · Services (card fan-out) · Packages (grid, magnetic CTA) ·
Contact (magnetic CTA) · Footer (full-screen cover flip).

Every heavy animation has a `prefers-reduced-motion` fade fallback and a touch
fallback (custom cursor + magnetic disabled, hero autoplay, native swipe).

## Swapping in real assets

Placeholders are generated by `npm run gen:assets`. Replace the files in
`/public/images`, `/public/frames/about`, `/public/videos` with real assets
using the **same filenames** (see naming in the script) and re-run
`npm run gen:assets` only if you want to regenerate the blur map. Asset
guidelines: masters ~2560px longest edge, sRGB, JPEG q85–90; full-screen
delivery < ~300–500KB (next/image emits WebP/AVIF automatically); frame
sequences 1280–1920px, heavily compressed.

> `ffmpeg` was not on PATH at generation time, so `/public/videos/*.mp4` are not
> present — the `<video>` posters render in their place. Drop real `.mp4`s in
> (or install ffmpeg and re-run `gen:assets`) to enable playback.
